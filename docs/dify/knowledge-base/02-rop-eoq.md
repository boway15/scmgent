# ROP 与 EOQ 说明

## ROP（Reorder Point，补货触发点）

```
ROP = 平均日销量 × 采购交期（天）+ 安全库存
```

当**有效供给**（可售 + 在途 + 在产池按规则计入）低于 ROP 时，系统生成补货建议或缺货预警。

## 安全库存

基于近 90 天销量标准差与交期计算，服务水准默认 95%（Z=1.65）。

## EOQ（经济订货量）

```
EOQ = √(2 × 年需求 × 订货成本 / 单位持有成本)
```

补货预测任务将 EOQ 作为建议补货量基准，并受 MOQ（最小起订量）约束抬升。

## 有效供给

- **单仓有效** = 可售 + 指向该仓的在途
- **仓网有效** = 区域内各仓有效之和 + SKU 在产池（未分仓）

## 亮灯规则

| 灯色 | 含义 |
|------|------|
| 红灯 | 低于 ROP 必须补货 |
| 黄灯 | 仅当同 SPU 有红灯 SKU 需补货时才生成建议 |
| 绿灯 | 不参与自动补货建议 |
