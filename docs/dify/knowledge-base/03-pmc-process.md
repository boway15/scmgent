# PMC 需求计划流程

## 定义

PMC 需求计划是平台向商家下发的 **SKU × 数量 × 交期** 计划（不含 BOM）。

## 标准流程

1. **补货建议** → 运行补货预测 → 在「补货建议」采纳行
2. **合并草稿** → 同商家 + 目标仓合并为 PMC 草稿计划
3. **计划列表** → 确认前可导出 CSV 发给商家
4. **确认计划** → 生成内部采购跟单台账

## 状态

`draft` → `confirmed` → 跟单 `open` / `closed`

## 与采购跟单的关系

采购跟单是计划确认后的**内部履约台账**，不是正式 ERP 采购单。用于标记是否已跟进商家。

## 常见操作

- 导出 CSV：计划列表 → 导出
- 查看跟单：下单计划 → 采购跟单
