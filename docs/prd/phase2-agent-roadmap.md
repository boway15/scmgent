# 阶段二/三智能体路线图

阶段一验证补货与 PMC 闭环后，按依赖关系扩展其余智能体。运营类智能体依赖销售平台 API，放在阶段二末或阶段三。

## 阶段二（3-6 个月）

### 采购智能体

| 功能 | 依赖数据 | 触发条件 |
|------|----------|----------|
| 采购单模板生成 | merchants、sku_suppliers、pmc_plans | PMC 计划 confirmed |
| 采购合同起草 | 合同模板库、历史 PO | 工厂回函 / 人工触发 |
| 工厂交付跟进增强 | purchase_drafts、跟进记录 | 已有 T-30/T-14/T-7 提醒 |

### 质检智能体

| 功能 | 依赖数据 | 触发条件 |
|------|----------|----------|
| 验货提醒 + Checklist | SKU 品类、质检标准库 | 采购交期 T-7 |
| 质检报告辅助 | checklist 结果、现场照片 | 质检员提交 |
| 质量问题库 | 历史缺陷记录 | 每次验货完成 |

### 物流智能体

| 功能 | 依赖数据 | 触发条件 |
|------|----------|----------|
| 船期推荐 | 组柜计划、货代 CSV 报价 | 调拨方案确认 |
| 订舱委托书 | 模板、货物描述 | 船期确认 |
| 提货提醒 | 截关时间 | 订舱确认 |

前置：阶段一 PMC 组柜/调拨方案（F1-002/F1-003）需先落地基础字段。

### 仓库智能体

| 功能 | 依赖数据 | 触发条件 |
|------|----------|----------|
| 入库预报 | 出港日期、航行天数、调拨方案 | 订舱确认 |
| 库存水位监控增强 | 覆盖天数模型（阶段一已有） | 每日 Cron |
| 滞销 SKU 识别 | 销量趋势、覆盖天数 | 每周一 |

### 平台集成

| 集成 | 说明 |
|------|------|
| 飞书审批流 | 补货确认、计划确认、合同审批 |
| 飞书 Webhook 全链路 | PMC → 采购 → 质检 → 物流 → 仓库事件驱动 |
| 自研系统试点 | 选 1-2 个关键接口（如 WMS 库存） |

## 阶段三（6-12 个月）

### 运营智能体（需销售平台 API）

| 功能 | 数据来源 | 触发 |
|------|----------|------|
| 竞品价格监控 | 手动 ASIN 列表 + 爬虫/API | 每日 |
| 差评分析 | Amazon SP-API Reviews | 新增 1-2 星 |
| 广告周报 | Amazon Advertising API | 每周一 |
| Listing 优化 | Listing 内容 + 差评关键词 | 人工触发 |

阶段一继续使用飞书多维表格维护销量；阶段三再替换为平台 API 自动拉取。

### 高级供应链

- 多仓自动调拨建议（基于仓网覆盖天数）
- BOM / MRP 物料缺口
- 供应商评分与质量季报
- 报关单证与合规审查（已移除的 compliance 模块可重启）
