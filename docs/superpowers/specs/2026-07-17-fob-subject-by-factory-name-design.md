# FOB 分账主体改为工厂名称 · 设计说明

**日期**：2026-07-17  
**状态**：已实现  
**关联**：`docs/prd/fob-settlement-split-v2.md`、`.cursor/rules/fob-settlement-dev.mdc`  
**触发案例**：柜号 `CSNU8685863` — 同订舱多工厂 FOB，平账仅显示 1 个主体

---

## 1. 背景与问题

FOB 分账的核心目标：把拖车/货代账单拆到不同**工厂/主体**，确认每厂承担金额。

截单清单体积导入当前把 `merchantCode` 设为**订舱编号**，工厂名称只进 remark / 展示名。同订舱、多工厂名称的货柜会被聚合成 1 个分账主体（例：`CSNU8685863` 三厂体积合计 69.22，平账显示「1 工厂/主体 · ZKX2604290075」）。

这与 PRD「按法人主体（公司）拆账」及业务口径不一致。

---

## 2. 决策摘要

| 项 | 决策 |
|----|------|
| 主体唯一键 | Excel「工厂名称」原文（trim），不做别名合并 |
| 方案 | 解析改键 + 平账展示强化（方案 2） |
| 工厂名为空 | 该行报错，不入体积（不回退订舱号） |
| ED / 简易 CSV | 键逻辑不改（已按主体列） |
| 历史批次 | 不自动迁移；需重导体积并重新核算 |

---

## 3. 主体定义与体积解析

### 3.1 截单清单（`parseTransferVolumeSheet`）

| 字段 | 新取值 |
|------|--------|
| `merchantCode` | 工厂名称 |
| `merchantName` | 工厂名称（与 code 相同） |
| 订舱编号 | 仍写入 remark `业务编号:…`，供柜级业务编号聚合 |
| 调拨 / 类别 / 目的仓 | remark 约定不变 |

- 同柜同厂名多行：体积相加；票数仍为柜内每主体 1 票
- 同柜不同厂名：多个分账主体

### 3.2 其他格式

- ED / 简易 CSV：维持现有「主体列 → merchantCode」逻辑
- 开发记忆（`fob-settlement-dev.mdc`）与导入 hint：改为「截单清单主体 = 工厂名称」

### 3.3 预期效果（`CSNU8685863`）

重导后应拆为三个主体，例如：

- 漳州贝德家具有限公司（约 37.70）
- 漳州悦淳工贸有限公司（约 31.29）
- 贝德家具有限公司（约 0.23）

合计体积仍约 69.22。

## 4. 平账 UI / 展示

### 4.1 平账矩阵（`FobContainerMatrixPanel`）

| 位置 | 现状 | 目标 |
|------|------|------|
| 柜号副标题 | `N 工厂/主体 · {业务编号}` | 主信息为 `N 工厂/主体`；有订舱时再跟 `· 业务编号 …`（可截断） |
| 悬停 tooltip | 柜号 / 业务编号 / 主体数 | 增加工厂名称列表（逗号分隔）；业务编号保留 |
| 展开行主体列 | 依赖 merchantName | 解析改后自然显示工厂名；不再把订舱号当主体 code |

### 4.2 分摊基数表（`ImportedVolumeStatsList`）

- 「工厂/主体」：只显示工厂名称；`merchantCode === merchantName` 时不括号重复 code
- 「业务编号」：仍按柜聚合订舱号（不变）

### 4.3 文案与导出

- 体积导入 hint：写明截单清单按**工厂名称**作为分账主体
- 导出列名不变；内容改为真实工厂名；ZIP 按厂分文件
- 本期不新增订舱号导出列

### 4.4 不改范围

- 分摊引擎（按体积 / 按票 / 固定 / 人工）
- 付款状态、服务商、拖车/货代账单解析

## 6. 测试与上线

### 6.1 单测

- 截单清单：`merchantCode` = 工厂名称
- 同柜多厂名 → 多个主体
- 工厂名为空 → 报错
- 贴近 `CSNU8685863`：同订舱三工厂 → 3 个 `merchantCode`

### 6.2 手工验收

1. 重导体积 → 分摊基数多家工厂，体积合计正确  
2. 重新核算 → 平账 `N 工厂/主体`，各厂有占比与承担金额  
3. 按工厂/主体导出 ZIP → 文件名对应各厂  

### 6.3 上线

- 已按订舱号核算的批次必须重导体积并重新核算
- Docker 改代码后 rebuild；浏览器强制刷新

### 6.4 范围外

- 工厂名称别名合并
- 历史批次自动改写
- 新增订舱号导出列

---

## 7. 成功标准

- 同订舱、多工厂名称的 FOB 柜，平账主体数 = 去重后的工厂名称数
- 各厂承担金额按该厂体积（及票规则）分摊，柜级总额仍平账
- UI 以工厂名为第一视觉，订舱号为辅助信息
