# 跨境资讯重构设计

## 1. 背景与目标

现有跨境资讯模块已具备 RSS/RSSHub 采集、去重、相关度评分、Dify 摘要和飞书多维表格同步能力，但现行分类只有“法规政策、活动运营、物流仓储、AI 提效、市场”，英文信源默认排除，且无法按家具业务、重点品牌和部门进行精确归类。

本次重构目标是建立一个面向家具跨境业务的每日资讯采集系统：

- 从多层、可维护的公开信源采集最新资讯。
- 自动筛选与公司业务相关的资讯，控制无效信息和重复报道。
- 对外文官方一手内容输出简体中文标题与摘要，同时保留原文链接。
- 以单一飞书多维表格作为资讯总表，使用主题、部门、国家、平台、品牌等多标签供业务人员按视图浏览。

本期仅涵盖采集、处理、存储和同步到飞书多维表格；明确不实现群机器人、消息推送、每日简报或 AI 行动建议。

## 2. 业务画像与优先级

### 2.1 主营业务

- 家具家居：沙发、桌子、椅子、床、升降桌等。
- 平台：Amazon、TikTok Shop、Wayfair、Walmart、独立站。
- 物流：海运、海外仓；物流资讯还须覆盖海关、清关、关税与贸易政策。
- 目标市场优先级：美国 > 英国、德国 > 法国、意大利。

### 2.2 重点品牌

以下品牌为竞品或行业标杆；仅跟踪公开新闻、品牌公告和行业媒体报道，不采集商品页价格、促销、库存或上架变动：

- Tribesigns、FEZIBO、Vernal、SONGMICS、FlexiSpot
- Eureka Ergonomic、Homary、Costway、POVISON、IKEA
- Ashley Furniture、Secretlab、Bestier、Yoobure、Hernest

品牌命中资讯应标记品牌标签，并优先关联产品开发、平台运营、营销推广或视觉设计等部门。

## 3. 范围

### 3.1 本期范围

1. 新建“跨境资讯总表”，旧“跨境电商新闻采集表”只保留历史记录。
2. 每日执行一次采集任务。
3. 接入并维护分层公开信源。
4. 抓取、语言识别、翻译、去重、过滤、分类、相关度评分和飞书同步。
5. 保存采集与同步日志，支持失败重试及问题追溯。
6. 支持飞书多维表格按部门、主题、国家/区域、平台、品牌和重要度建立视图。

### 3.2 不在本期范围

- 飞书群消息、机器人、定时推送或日报。
- 人工审核作为入表前置条件。
- AI 生成业务影响、建议动作或决策结论。
- 商品页爬取、价格/促销/库存监控。
- 付费墙、登录态、反爬限制或受版权限制全文的绕过。

## 4. 分层信源策略

### 4.1 一级：官方一手来源

用于法规、平台规则、物流与重大外部事件的准确性保障：

- 目标市场的贸易、海关、税务、产品安全、环保与消费者保护主管机构。
- Amazon、TikTok Shop、Wayfair、Walmart 的卖家公告、政策中心或官方新闻室。
- 航运公司、港口、海关、物流服务商、海外仓服务商的公开公告。
- 中国海关总署、商务部等与出口贸易直接相关的官方渠道。

英文一级来源允许入库；系统将翻译为简体中文，并保留原文标题、原文链接和“官方来源”标识。

### 4.2 二级：中文垂直媒体

用于补充跨境电商、家具家居、供应链物流、营销推广和 AI 的行业背景。每条关键事实应尽可能附带其原始来源链接或可核验来源。

### 4.3 三级：经验证的聚合信源

RSSHub 或其他稳定聚合源仅用于发现线索。内容仍需通过更严格的业务相关性、来源可信度和重复检查后才能入表。

### 4.4 信源配置

每个信源至少维护：

- `code`、名称、主页/订阅 URL、采集类型（RSS、RSSHub、公告页）。
- 信源等级（一级/二级/三级）、是否官方、语言、启停状态。
- 适用主题、平台、国家/区域、品牌及部门范围。
- 最近成功采集时间、连续失败次数、错误信息。

## 5. 处理链路

```text
每日调度
  → 拉取信源新增内容
  → 正文抽取与发布时间校验
  → 语言识别
  → 英文官方内容翻译为简体中文
  → URL/标题/正文相似度去重
  → 业务相关性硬过滤
  → 主题、部门、平台、国家、品牌、业务标签分类
  → 相关度与重要度计算
  → 本地持久化
  → 幂等同步到飞书跨境资讯总表
  → 记录每信源任务日志
```

### 5.1 相关性硬过滤

资讯至少命中以下一个业务锚点，才进入进一步分类：

- 家具家居品类、材质、工艺、设计趋势或重点品牌。
- 目标平台及卖家政策、运营规则、广告或营销动态。
- 美国、英国、德国、法国或意大利的市场、法规或消费动态。
- 海运、港口、清关、关税、海外仓、尾程物流等供应链事项。
- 与目标业务直接相关的 AI 应用、视觉设计或营销技术。

以下内容直接过滤：泛电商且没有业务锚点的资讯、与目标市场无关的地区新闻、纯广告、纯旅游/娱乐内容，以及重复报道。

### 5.2 中文化原则

- 中文信源：保留原始中文标题并生成压缩摘要。
- 英文一级官方信源：保留原文标题，生成事实忠实的简体中文标题与摘要。
- 摘要不得补充原文没有明确说明的影响判断、推测或行动建议。

### 5.3 去重原则

- 同一规范 URL：直接去重。
- 不同 URL 的转载或同事件报道：根据规范化标题、发布时间窗口和正文相似度合并。
- 多来源报道同一事件时，保留官方或信源等级更高的条目，并可在来源追溯字段记录其他报道。

## 6. 分类体系

### 6.1 主题分类（单选主分类）

1. 产品开发与家具趋势
2. PMC 与供应链
3. 采购与供应商
4. 物流、海关与关税
5. 平台运营
6. 营销推广
7. 视觉设计
8. AI 前沿
9. 法规与外部环境

### 6.2 相关部门（多选）

- 产品开发
- PMC
- 采购
- 物流
- 平台运营
- 营销推广
- 视觉设计
- AI
- 法规与外部环境

一条资讯可归属多个部门。例如美国关税政策可同时关联物流、采购、PMC 和法规与外部环境。

### 6.3 业务标签（多选）

- 平台：Amazon、TikTok Shop、Wayfair、Walmart、独立站。
- 国家/区域：美国、英国、德国、法国、意大利及欧盟。
- 物流环节：海运、港口、清关、关税、海外仓、尾程。
- 品类：沙发、桌子、椅子、床、升降桌及后续扩展项。
- 品牌：第 2.2 节列出的重点品牌。

## 7. 飞书多维表格设计

新表名称：`跨境资讯总表`。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 中文标题 | 文本 | 业务人员阅读的主标题 |
| 中文摘要 | 多行文本 | 事实型简体中文摘要 |
| 原文标题 | 文本 | 外文或原始标题，用于追溯 |
| 原文链接 | 超链接 | 文章来源 |
| 信源名称 | 单选/文本 | 来源名称 |
| 信源等级 | 单选 | 一级、二级、三级 |
| 官方来源 | 复选框 | 是否为官方一手来源 |
| 原文语言 | 单选 | 中文、英文等 |
| 发布时间 | 日期时间 | 原文发布时间 |
| 采集时间 | 日期时间 | 系统发现时间 |
| 主题分类 | 单选 | 第 6.1 节分类 |
| 相关部门 | 多选 | 第 6.2 节分类 |
| 平台标签 | 多选 | 平台关联 |
| 国家/区域标签 | 多选 | 市场关联 |
| 业务标签 | 多选 | 品类与物流环节等 |
| 品牌标签 | 多选 | 重点品牌 |
| 相关度评分 | 数字 | 0–100 |
| 重要等级 | 单选 | 高、中、低 |
| 筛选命中依据 | 多行文本 | 关键词、实体和规则命中，供追溯 |
| 业务有效性 | 单选 | 有效、无效、误分类；供后续规则调优 |
| 系统文章 ID | 文本 | 本地记录 ID，支持幂等同步 |

建议预建视图：全部最新资讯、各部门、美国/英国德国/法国意大利、各平台、物流与关税、重点品牌、官方资讯和高相关度资讯。

## 8. 准确性与可观测性

本期采用全自动入表，不设人工审核阻塞。因此系统不能承诺零误报或零漏报；质量通过以下机制控制：

- 一级官方来源的可信度权重最高；二、三级来源作为补充。
- 业务锚点硬过滤，避免泛行业噪音。
- 信源等级、相关度、命中依据、原文链接和系统文章 ID 全部写入，保障可追溯。
- 每日记录信源的抓取量、新增量、过滤量、去重量、失败原因和飞书同步状态。
- 单个信源失败不阻断其他信源；飞书写入失败进入待同步状态并可重试。
- 在飞书中预留业务有效性字段（有效/无效/误分类）供业务人员标记，作为后续调整词库、信源权重和阈值的依据，但不影响本期自动入表。

## 9. 与现有模块的差异

现有实现需要在实施阶段处理的关键差异：

- 现有策略要求中文内容并排除了英文域名；重构后应仅允许英文一级官方来源，并走翻译流程。
- 现有飞书字段只有标题、概要、来源、链接、分类、结构化汇总和备注；重构需映射到第 7 节的新表字段。
- 现有分类与部门标签不足以覆盖家具、重点品牌、TikTok Shop、独立站、法国、意大利及九类部门。
- 现有任务可按信源 8–24 小时间隔执行；重构为每日一次的统一调度策略。
- 旧飞书表仅保存历史；新表须使用独立的表标识和同步配置，避免改变旧记录。

## 10. 验收标准

1. 每日任务能够从已启用信源采集新增内容，并写入新飞书总表。
2. 入表记录均包含简体中文标题、简体中文摘要、原文链接、信源、发布时间、主题分类和相关部门。
3. 英文官方公告可被保留并形成含原文追溯信息的中文记录；非官方英文媒体默认不入表。
4. 家具、重点平台、目标国家、海运/海外仓、海关/关税及重点品牌资讯能够被正确标记。
5. 同一 URL 或同一事件的转载不会产生重复记录。
6. 对任意记录可从飞书字段追溯信源等级、原文、相关度与筛选命中依据。
7. 对单个信源抓取失败或飞书同步失败，任务日志可查询失败原因，其他信源不受影响。
8. 系统不发送飞书群消息、不生成行动建议，也不抓取商品价格、促销或库存。
